# 报告格式指南

## 周报结构

### 1. 摘要（2-3 句话）
简要概述本周重点工作。

### 2. 已完成任务
- 使用要点列表
- 尽量包含可量化的结果
- 示例："完成 API 集成，响应时间降低 30%"

### 3. 进行中的工作
- 当前任务及预计完成日期
- 如适用，标注完成百分比

### 4. 阻塞项与风险
- 可能延误进度的问题
- 对其他团队的依赖

### 5. 下周计划
- 按优先级排列的计划任务
- 关键里程碑

## 格式规范
- 使用 Markdown 标题划分章节
- 每个要点保持一行
- 日期使用 YYYY-MM-DD 格式
- 结构化数据使用表格呈现
